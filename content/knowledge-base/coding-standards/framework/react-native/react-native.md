- Keep screens focused on orchestration and move reusable logic into hooks or services.
- Isolate platform-specific branches behind small adapters or components.
- Keep navigation contracts explicit and typed.
- Use TypeScript strict mode.
- Compose application-wide concerns through root providers for configuration, auth, logging, API client, query state, notifications, location, and similar cross-cutting concerns.
- Keep server-state fetching and invalidation behind dedicated hooks or query abstractions.
- Separate secure storage for secrets from cache storage for offline and local data.
- Initialize telemetry and crash reporting conditionally by environment and fail fast on invalid production-like configuration.
- Treat environment selection and environment-specific app configuration as explicit runtime inputs.
- Handle upload, media, and network timeouts explicitly.
- Distinguish OTA-safe changes from changes that require a native rebuild.
- Prefer functional components and hooks.
- Prefer thin UI components over typed state and domain actions.
- Prefer explicit loading, empty, and error states for async views.
- Prefer provider composition at the app root.
- Prefer React Query or equivalent for server state and cache invalidation.
- Prefer dedicated service modules for API access and platform integrations.
- Prefer dedicated components for complex media behavior.
- Prefer reusable theme modules for colors, typography, spacing, and shared styles.
- Prefer cached read models for degraded or offline usage where the product needs it.
- Do not embed business logic directly in screen components.
- Do not use unbounded global state for local screen concerns.
- Do not use implicit navigation params without typed contracts.
- Do not mix secure secrets and general cache storage in the same persistence mechanism.
- Do not scatter network access and storage access directly across screens.
- Do not allow silent production telemetry misconfiguration.
